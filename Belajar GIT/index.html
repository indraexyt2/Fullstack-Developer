Git Commands

Create

  • git init: Menginisialisasi repository Git baru di direktori saat ini. Membuat folder .git yang berisi semua file metadata untuk repository.
  • git clone <repository-url>: Menggandakan repository Git yang sudah ada dari URL tertentu ke direktori lokal.
    • --branch <branch-name>: Mengklon branch tertentu.
    • --depth <depth>: Mengklon repository dengan kedalaman tertentu (misalnya hanya commit terbaru).

Make a Change

  • git add <File>: Menambahkan perubahan pada file tertentu ke staging area.
    • git add .: Menambahkan semua perubahan di direktori saat ini ke staging area.
    • git add -A: Menambahkan semua perubahan (baik tracked maupun untracked) ke staging area.
  • git commit -m "<Message>": Membuat commit baru dengan pesan tertentu.
    • git commit -a -m "<Message>": Menambahkan semua perubahan tracked files dan membuat commit dengan pesan tertentu.
  • git reset <File>: Menghapus file tertentu dari staging area tanpa menghapus perubahan pada working directory.
    • git reset --soft <commit>: Mengatur ulang ke commit tertentu, menyimpan perubahan pada staging area.
    • git reset --hard <commit>: Mengatur ulang ke commit tertentu dan menghapus semua perubahan pada working directory.

Observe

  • git status: Menampilkan status dari working directory dan staging area.
    • -s atau --short: Menampilkan status dalam format yang lebih ringkas.
  • git diff: Menampilkan perubahan yang belum di-stage.
    • git diff <branch>: Menampilkan perbedaan antara branch saat ini dengan branch tertentu.
  • git show <commit>: Menampilkan detail dari commit tertentu.
    • git show <commit>:<file>: Menampilkan perubahan pada file tertentu di commit tertentu.
  • git log: Menampilkan riwayat commit.
    • --oneline: Menampilkan log dalam format satu baris per commit.
    • --graph: Menampilkan log dalam format grafis.

Sync

  • git remote add origin <repository-url>: Menambahkan remote repository dengan nama origin.
    • git remote -v: Menampilkan daftar remote repository yang ada.
  • git push --set-upstream origin <branch>: Mengirimkan branch lokal ke remote repository dan mengatur upstream tracking.
    • git push -u origin <branch>: Singkatan dari perintah di atas.
  • git push: Mengirimkan commit dari branch lokal ke remote repository.
    • --force: Memaksa push yang dapat menimpa perubahan di remote repository.
  • git pull: Mengambil dan menggabungkan perubahan dari remote repository ke branch lokal.
    • --rebase: Menggunakan rebase alih-alih merge.
  • git fetch: Mengambil perubahan dari remote repository tanpa menggabungkannya ke branch lokal.
    • --all: Mengambil perubahan dari semua remote repository yang dikonfigurasi.

Branch

  • git branch: Menampilkan daftar branch atau membuat branch baru.
    • -d <branch>: Menghapus branch tertentu.
    • -m <new-name>: Mengganti nama branch saat ini.
  • git checkout <branch>: Berpindah ke branch tertentu.
    • -b <branch>: Membuat branch baru dan langsung berpindah ke branch tersebut.
  • git merge <branch>: Menggabungkan branch tertentu ke branch saat ini.
    • --no-ff: Menghindari fast-forward merge.
  • git tag: Menambahkan tag pada commit.
    • -a <tag> -m <message>: Membuat annotated tag dengan pesan tertentu.
    • -d <tag>: Menghapus tag tertentu.
